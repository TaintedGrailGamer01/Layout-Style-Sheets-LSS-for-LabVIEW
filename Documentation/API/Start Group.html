LSS.lvlib:Start Group.vi

Creates groupings of objects based on the Style Sheet input. All styles applied to the group itself will apply to all objects of the group.

Groupings must be started before tags are marked into controls. Otherwise the groups will not be marked into the tag hierarchy properly.

There can be any number of groups, but each control can presently only belong to one group.

 error in (no error)

error in can accept error information wired from VIs previously called. Use this information to decide if any functionality should be bypassed in the event of errors from other VIs.

Right-click the error in control on the front panel and select Explain Error or Explain Warning from the shortcut menu for more information about the error.

Front Panel Objects

Contains array of all objects on the front panel.

    Generic reference to a front panel object.

Style Sheet

Style sheet determining how objects will be grouped. The format of the style sheet for grouping objects is as follows:

grouping{

objectGroupName:frontPanelObjectName;

objectGroupName2:frontPanelObjectName2;

}

Where the specifier grouping must stay the same. objectGroupName is the name of the group, for example "Group 1". frontPanelObjectName is the name of the item on the front panel, for example "Stop Button". To add two buttons to group alpha would look as follows:

grouping{

group alpha:Start Button;

group alpha:Stop Button;

}

error out

Returns errors from this VI if any occured.
