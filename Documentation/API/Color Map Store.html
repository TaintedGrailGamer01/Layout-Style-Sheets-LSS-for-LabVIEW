LSS.lvlib:Color Map Store.vi

This functional global variable stores the color map values so other LSS functions can make sure of those values without needing to pass in the color map values with each LSS string.

 colorMap

This string is an LSS formatted string that allows the mapping of hexidecimal color constants to string names that are easier to remember. Once a color is defined in the colorMap, any subsequent LSS call can simply use the name of the color instead of the hexidecimal representation. This input is only used when the mode input is set to "Write" or "Add". The format for this string is as follows:

colorMap{

myColorName:#000000;

}

Mode

Select between the following options:

1. Write - Saves the colorMap input string for use by later LSS functions. Will overwrite any previously saved colormap values.

2. Add - Adds the new colorMap LSS string to the existing string.

3. Read - Reads all currently saved color map values.

Error In

error in can accept error information wired from VIs previously called. Use this information to decide if any functionality should be bypassed in the event of errors from other VIs.

Right-click the error in control on the front panel and select Explain Error or Explain Warning from the shortcut menu for more information about the error.

colorMap

Contains the data from the colorMap input string as a parsed array that is used by LSS functions for changing style of the UI.

    Array of color map name value pairs.

      		color-name

      Name of the color as a string

      		hex-value

      Value of the color in hex

colorMap String

Outputs the text of the items saved in the color map as a string.

Error out

Returns errors from this VI if any occured.
